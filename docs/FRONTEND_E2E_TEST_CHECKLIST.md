# 前端端到端测试任务清单

> 用于在前端启动后逐项手工测试各功能，便于记录通过/失败与备注。  
> **前置条件**：后端已启动（如 `python run.py` 监听 5001），前端 `npm run serve`（默认 8080），`.env.development` 中 `VUE_APP_API_BASE_URL=http://localhost:5001`。  
> **建议**：手工测试前先跑一遍前后端 API 对接检查：在 knowledge_gragh 项目根目录执行 `pip install requests` 后运行 `python scripts/check_frontend_backend_api.py`，确认 19/19 通过再按清单逐项测试。

## 六、数据上传（/upload）

> **说明**：`/upload` 需登录，未登录会被重定向到 `/login`。请先完成 2.2 登录。

| 序号 | 项目 | 操作步骤 | 预期 | 结果 | 备注 |
|:---:|------|----------|------|:----:|------|
| 6.1 | 进入上传页 | 点击「数据上传」或访问 `/upload` | 上传区、文件列表、文档知识库区块、Hadoop 批量操作区、数据模板区正常展示 | ☐ | |
| 6.2 | 文件上传 | 选择或拖拽 1 个 TXT/PDF/JSON/CSV（≤50MB），点击「开始上传」 | 请求 `POST /api/upload`；进度条更新；完成后列表中显示「上传成功」及 `fileId` | ☐ | 若后端无 `/api/upload`，会报错，记录状态码与文案 |
| 6.3 | 单个构建图谱 | 在已完成文件中点击「生成知识图谱」 | 请求 `POST /api/kg/build`（body: `{ file_id }`）；轮询 `GET /api/kg/build/progress/{id}`；进度条与「已生成」状态更新 | ☐ | |
| 6.4 | 文档知识库-刷新列表 | 在「文档知识库」区块点击「刷新文档列表」 | 请求 `GET /api/kb/documents/list`；表格展示已入库文档或空态 | ☐ | |
| 6.5 | 文档知识库-入库 | 在文件列表中点击某文件「入库」 | 请求 `POST /api/kb/documents/ingest`（`file_id`）；成功后该文件显示「已入库」、块数；刷新列表后可见 | ☐ | |
| 6.6 | 文档知识库-批量入库 | 选择多个已上传未入库文件，点击「批量入库」 | 对每个 `file_id` 调用 `ingest`；列表中对应项变为「已入库」 | ☐ | |
| 6.7 | 文档知识库-删除 | 在文档列表中对某条点击「删除」 | 请求 `DELETE /api/kb/documents/{doc_id}`；该条从列表移除或刷新后消失 | ☐ | |
| 6.8 | 文档知识库-重索引 | 对支持重索引的文档点击「重索引」 | 请求 `POST /api/kb/documents/reindex`（body 含 `doc_id`）；有成功/失败提示 | ☐ | 部分 doc 可能不可重索引，按 UI 禁用逻辑 |
| 6.9 | Hadoop-批量上传 HDFS | 选择已上传、未传 HDFS 的文件，点击「批量上传至HDFS」 | 请求 `POST /api/hadoop/upload/batch`；有进度或结果反馈 | ☐ | 依赖 Hadoop 与 Celery 环境 |
| 6.10 | Hadoop-批量构建 | 选择已上传文件，点击「批量构建知识图谱」 | 请求 `POST /api/hadoop/build/batch`；获得 `task_id`；轮询 `GET /api/hadoop/status/{id}`；进度与状态展示 | ☐ | 依赖 Hadoop/Celery |
| 6.11 | 数据模板下载 | 点击任一模板按钮 | 触发下载，无报错 | ☐ | 若为前端静态文件则易通过 |

## 七、知识图谱（/graph、/graph/:file_id）

> **说明**：图谱页会请求 `GET /api/graph/list` 与 `GET /api/graph/data`（可选 `graph_id`）。`/graph/:file_id` 已支持；放大/缩小通过 zrender getRoots 实现；筛选含疾病/症状/治疗方法/药物/检查/部位；导出为「完整数据 JSON」或「当前视图图片」；「返回」清空搜索与筛选并恢复完整视图；节点弹窗「问医生」跳转问答页并自动发送「请为我介绍一下XXX」。

| 序号 | 项目 | 操作步骤 | 预期 | 结果 | 备注 |
|:---:|------|----------|------|:----:|------|
| 7.1 | 进入图谱页 | 点击「知识图谱」或访问 `/graph` | 请求 `GET /api/graph/list`；图谱下拉、画布、缩放/导出/返回/图例等控件正常 | ☐ | |
| 7.2 | 图谱列表与切换 | 在下拉中选择不同图谱或「全部图谱」 | 请求 `GET /api/graph/data`（带 `graph_id` 或不带）；画布展示节点与关系，或空态 | ☐ | |
| 7.3 | 带 file_id 进入 | 访问 `/graph/{file_id}`（`file_id` 为已构建图谱的文件 ID） | 自动选中该图谱并请求 `GET /api/graph/data?graph_id={file_id}`；后端按 file_id 解析，画布展示或 404 | ☐ | |
| 7.4 | 搜索、筛选、导出、返回 | 使用实体搜索、类型筛选、导出（JSON/图片）、返回、图例 | 搜索/筛选为前端过滤；导出为完整 JSON 或当前视图图片；返回恢复完整视图；图例切换显示/隐藏 | ☐ | |
| 7.5 | 问医生 | 点击节点→弹窗内点击「问医生」 | 跳转 `/chat`，自动填入并发送「请为我介绍一下XXX」（XXX 为节点名） | ☐ | ChatView 根据 query.question+autoSend 处理 |

## 使用说明

1. **结果列**：通过打 `✓`，失败打 `✗`，并可在「备注」中写原因或现象。  
2. **依赖后端**：若某接口后端未实现或未启动，该条预期为「明确错误提示」或 4xx/5xx，可备注「后端未实现」后再继续其它项。  
3. **推荐顺序**：按章节从 一 → 十二 执行；其中 二（登录）建议最先完成，以便测试 六、九、十、十一。
