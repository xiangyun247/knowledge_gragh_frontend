# 智护银龄·忆路康（前端）

> Vue 2 + Element UI 的适老化原型前端：围绕“低认知负荷”设计，覆盖老人核心场景（首页三件小事 / 服药提醒 / 问一问 / 学知识），并提供紧急求助（SOS）、全局语音入口、认知负荷评估闭环（行为埋点 + 问卷 + 趋势图）、家属周报与机构看板。对应大创主题：**智护银龄 —— 基于 AI 与认知负荷的老龄认知障碍人群低认知负荷知识辅助**。

## 项目简介

本前端为「智护银龄·忆路康」老龄认知障碍低认知负荷知识辅助系统的用户界面，主要能力包括：

- **智能问答**：对接后端 Agent 流式接口，支持多轮对话、关联来源（图谱/文献）展示与深度思考推理过程
- **患者教育与科普推文**：在聊天页中支持「患者教育」与「科普推文」两种模式，可一键生成给患者看的说明书或短篇科普文案
- **知识图谱**：多图谱切换、实体搜索与筛选、节点详情、导出与「问医生」跳转
- **实体搜索**：关键词与类型筛选、结果列表与详情、查看图谱关联
- **文档知识库**：个人/共享知识库、文献检索、文档列表、基于知识库提问（跳转聊天并自动发送）
- **数据上传**：文件上传、单文件/批量生成知识图谱、入库与 Hadoop 批量构建
- **历史记录**：问答/搜索/图谱查询/图谱构建/上传等多类型，筛选、分页、删除（同步后端）
- **服药提醒（P1）**：用药清单 + 今日任务 + 进度反馈，支持认知负荷埋点与问卷
- **认知负荷评估（P4）**：行为事件与问卷趋势图、波动报告、导出与上传到后端
- **EEG 实时监测**：受试者管理、会话创建与结束、实时数据可视化、历史记录与汇总统计
- **机构看板（P3）**：管理员/医生聚合统计与图表（全站用户维度）
- **家属周报（A1）**：本地行为与问卷汇总，生成周度照护摘要与建议
- **适老简化模式（C4）**：老人角色下仅保留四入口，路由自动防迷路；含小忆聊天页（ElderlyChatView）：3档字号、深色模式、TTS朗读、语音输入、快捷回复、情绪指示器、记忆面板、消息右键复制与重新生成、滚动到底部按钮、发送防抖
- **全局语音助手（C2）**：按住说话→识别意图（导航/提问）→语音反馈（TTS）
- **容错机制（C3）**：危险操作确认 + 全局返回条（返回 + 我在哪）
- **新手引导 & 情感化反馈（C5/C6）**：适老首次引导与完成任务正反馈
- **认知负荷实验测试流程**：受试者信息注册 → 基线问卷（以豆包/微信为参照）→ 小忆聊天（行为埋点采集）→ 后测问卷 → 认知负荷评分（基于NASA-TLX前后测delta+行为惩罚，非随机）
- **机构看板数据导出**：汇总页面支持一键导出全部数据CSV（概览/趋势/来源分布/事件类型/问卷得分/用户排行）

三主题（科技风 / 医疗风 / 适老模式）可切换并持久化；ECharts 图表会随主题切换自动重绘（颜色一致）。

- **认知负荷实验测试流程完善**：ElderlyTestView 支持受试者信息完整注册（编号/年龄/性别/认知状态/备注）、自动同步后端、基线问卷引导语改为以日常APP为参照（支持对照组对比设计）

- **问卷数据采集修复**：修复问卷答案未写入localStorage的bug，评估页面与汇总看板均能正确读取问卷数据
- **机构看板数据导出**：CognitiveDashboardView 新增「导出全部数据」CSV功能（6个Sheet：概览/趋势/来源分布/事件类型/问卷得分/用户排行）
- **EEG监测历史记录增强**：新增「清空全部」按钮（二次确认+逐条删除），解决逐条删除效率低问题
- **NASA-TLX适老化量表**：6维度1-5分emoji按钮选择，基线问卷引导语区分参照标准（日常APP vs 小忆体验）
- **小忆聊天页全面优化**：3档字号切换、浅色/深色/跟随系统三档主题、TTS朗读（智能选音优先微软晓晓）、语音输入60s限制、快捷回复按钮、情绪指示器动态计算、记忆面板从对话提取、右键复制+重新生成、滚动到底部浮动按钮
- **新增患者教育视图**：聊天页面支持结构化展示患者教育内容（标题 + 分节 + 温馨提示），并有单独的「患者教育」模式方便导师/评委演示。
- **多模态工具接入**：集成语音输入（STT）与图片解读（OCR+通俗解释），在聊天输入区提供麦克风与图片入口，与后端多模态接口打通。
- **智能体扩展模式**：在普通问答基础上增加「患者教育」与「科普推文」模式按钮，以及消息内「推文版」操作，UI 已适配新返回结构。
- **图文效果优化**：聊天右侧的「关联信息」区分「来自图谱」「来自文献」，并支持深度思考（检索与推理过程）折叠面板，整体图文结合效果更适合演示。
- **多模型选择器**：在聊天输入区新增模型下拉框，可在 DeepSeek、GPT、Gemini、通义、豆包、智谱、文心一言、Kimi、Grok 等模型间切换；未配置的模型在选择/发送时会弹窗提示「未配置该模型」。
- **Chat Markdown 渲染（B1）**：RAG/Agent 返回支持 Markdown 排版（列表、加粗、引用、表格、代码块等），更接近“专业系统”。
- **适老闭环增强（C7）**：首页“今日日程”支持完成/提醒中/错过/推迟，并可“稍后提醒”；时间点会根据使用习惯动态调整。

## 核心特性

### 双主题

- **科技风**：深色背景、霓虹青蓝与紫色强调

## 使用方式

### 环境要求

- Node.js >= 12
- npm >= 6 或 yarn >= 1

### 安装与运行

```bash
# 安装依赖
npm install

# 开发
npm run serve
# 或
npm run dev
```

默认访问：**http://localhost:8080**

> 说明：当前开发环境端口可能被占用时自动顺延（如 5003/5004/5005），以终端输出为准。

### 生产构建

```bash
npm run build
# 或
npm run build:prod
```

产物在 `dist/`，部署时需保证请求能转发到后端 API 基地址（见下文）。

### 环境变量与后端对接

- 开发环境：根目录 `.env.development`  
- 生产环境：`.env.production`

关键变量（根据环境区分）：

```bash
# 开发环境（.env.development）：本地联调指向后端
VUE_APP_API_BASE_URL=http://localhost:5001

# 生产环境（.env.production）：云上 Nginx 同源部署时通常留空，使用相对路径 /api/...
VUE_APP_API_BASE_URL=
```

前端通过 `src/utils/request.js` 封装的 axios 实例发起请求：

- 当 `VUE_APP_API_BASE_URL` 配置为 `http://localhost:5001` 等完整地址时，请求会直接发往该地址；
- 当生产环境留空时，使用相对路径（如 `/api/xxx`），由 Nginx 将 `/api` 转发到后端，避免 CORS 问题。

**联调建议：**

1. 先启动后端（如 `python run.py`，端口与上面一致）。
2. 前端 `npm run serve`，确认登录、问答、图谱、搜索、上传、历史等接口均指向同一后端。
3. 若后端在不同端口或域名，只改 `VUE_APP_API_BASE_URL` 并重启前端即可。

## 与后端对接说明

### 接口模块（src/api）

| 模块 | 说明 |
|------|------|
| `chat.js` | Agent 流式问答（`/api/agent/query/stream` 等） |
| `graph.js` | 图谱列表、图谱数据、清除（`/api/graph/list`、`/api/graph/data` 等） |
| `search.js` | 实体搜索（`/api/search/entities` 等） |
| `kb.js` | 知识库列表、文档列表、检索（`/api/kb/...`） |
| `upload.js` | 文件上传（`/api/upload`） |
| `history.js` | 历史列表、删除（`/api/history/list`、`/api/history/:id` 等） |
| `hadoop.js` | 批量上传 HDFS、批量构建知识图谱（Hadoop/Celery） |
| `auth.js` | 登录、注册等（若后端提供） |
| `eegSession.js` | EEG 会话管理（`/api/eeg-session/...`） |

历史写入由 `src/utils/historyUtils.js` 统一封装（调用上述 history 接口并落库）。

### 页面与接口对应关系

- **首页**：指标与图表当前为前端示意数据；**后续将对接后端统计接口**，用于总实体数、关系数、图谱数、检索次数、活跃度等。
- **智能问答**：依赖 Agent 流式接口与 session；返回中的来源（图谱/文献）用于右侧展示与跳转。
- **知识图谱**：依赖图谱列表与图谱数据接口；支持按 `graph_id` 或 `file_id` 加载。
- **实体搜索**：依赖实体搜索接口；详情内「查看关联实体」使用当前结果列表 `searchResults` 按 `targetId` 查找。
- **文档知识库**：依赖知识库列表、文档列表、检索接口；底部「基于知识库提问」跳转 `/chat?question=xxx&autoSend=1`，与聊天页约定一致。
- **数据上传**：依赖上传接口、图谱构建、知识库入库、Hadoop 等接口；上传与构建结果会写入历史。
- **历史记录**：依赖历史列表与删除接口；支持类型、状态、时间筛选与分页。
- **FAQ**：分类与题目当前为前端数据；**可改为对接后端 FAQ 接口**；**提交反馈计划接后端**。

## 常见问题

- **如何切换主题？** 顶栏右侧主题切换按钮，选择后会自动保存。
- **如何进入“老人简化模式”？** 进入 `/profile`，将角色切换为“老人（简化模式）”，系统会自动切换到适老主题并简化导航。
- **主题切换后图表颜色没变？** 已统一做了主题切换重绘；若仍异常，刷新页面即可（开发环境 HMR 偶发缓存）。
- **聊天页自动带出问题？** 使用链接参数 `?question=你的问题&autoSend=1`，例如从知识库「基于知识库提问」或图谱「问医生」跳转。
- **图谱/搜索/历史数据不更新？** 检查 `VUE_APP_API_BASE_URL` 与后端是否一致，以及网络与控制台报错。

更多使用问题可查看系统内 **常见问题** 页（`/faq`）。
